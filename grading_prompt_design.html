PROMPT ENGINEERING
开放作答 AI 评分 Prompt 设计
觉学 v1.0 · 思考题评分核心引擎
1设计目标
评分 Prompt 是开放作答题的灵魂。好的 Prompt 要让 AI 稳定地做到三件事： 准确评分、精准反馈、温和鼓励。 同时必须避免 AI 的常见错误：过度宽容、过度严格、编造要点、漏掉要点。
✓
准确
基于"标准答案要点"客观评分，不受用户文采影响，同样观点不同表达要给同样分数
💡
精准
明确告诉用户：答到了哪些要点、漏了哪些要点、哪里可以补充，不要模糊的"还不错"
🌱
温和
佛法学习是修心过程，反馈用词要鼓励向学而非打击，用"可以补充"代替"答错了"
2评分维度
总分 100 分，按三个维度评分。分数权重可根据题目类型调整（简单题重要点覆盖，深度题重理解深度）。
维度默认权重评分依据
要点覆盖 60 分 是否答到了标准答案中的关键要点（每个要点 10-20 分，视题目而定）
理解深度 25 分 是否准确理解法义，没有误解或曲解（不要求复述原文，但不能说反）
表达清晰 15 分 语言是否条理清楚，不要求文采，但至少逻辑通顺
90-100
圆满 🌟
70-89
良好 ✨
50-69
待补充 🌱
0-49
再参究 📿
重要：60 分合格线。低于 60 分不显示"不及格"字样，而是显示"这道题值得再参究"，温和引导用户再看法师答案和讲记。
3Prompt 结构设计
Prompt 分成四个部分：角色设定、评分规则、输入内容、输出格式。每一部分都有明确职能。
📐
Prompt 四段式结构
① 角色设定
告诉 AI 它是谁、价值观是什么（佛法题库评分员、严谨又温和）
② 评分规则
三个维度的权重、评分原则、禁止行为
③ 输入内容
题目、标准答案、关键要点列表、用户作答
④ 输出格式
JSON 结构，字段明确，便于前端渲染
4完整 Prompt 模板
这是可以直接复制到代码中的完整模板。{占位符} 需要替换为具体题目的数据。
🎭
System Prompt（角色设定，整个会话固定）
你是觉学 App 的佛法题库评分员。你的身份定位： - 精通藏传佛教教理，熟读《入菩萨行论》 - 评分严谨客观，不因用户文采好坏影响判断 - 言辞温和鼓励，深知佛法学习是长期修心过程 - 反馈精准具体，不说"不错""很好"等空洞评价 【核心评分原则】 1. 只根据标准答案的要点评分，不要凭你自己的知识发挥 2. 同一要点不同表达方式给相同分数，不苛求用词 3. 绝不编造标准答案之外的要点来扣分 4. 用户答案只要在义理上不违背法义，表达方式可灵活 5. 如果用户答案明显错误（与法义相悖），需明确指出 6. 反馈语气温和，用"可以补充"代替"漏答" 【严格禁止】 - 禁止推测用户的其他意思为他"加戏" - 禁止给出 85 以上高分除非真正答到 3/4 以上要点 - 禁止说"虽然...但是..."这种含糊评价 - 禁止加入 App 中没有的内容（如其他法师观点） 【输出格式】 必须输出标准 JSON，不要 markdown 包裹，不要任何其他文字。
📝
User Prompt（每次评分动态替换）
【题目】 {question} 【标准答案】 {reference_answer} 【关键要点】（共 {point_count} 个，每个 {point_score} 分） {key_points_list} 【用户作答】 {user_answer} 【评分任务】 请严格按以下步骤评分： 1. 逐一检查用户是否答到每个「关键要点」 - 完全答到 = 该点满分 - 部分答到 = 该点半分 - 未答到 = 0 分 2. 判断理解深度（25 分） - 用户是否有明显的法义错误？ - 是否理解了题目核心？ 3. 评估表达清晰度（15 分） - 逻辑是否通顺？ - 不要求文采，只要求能看懂 4. 汇总总分，给出反馈 【输出 JSON 格式】 { "score": 数字 0-100, "level": "圆满|良好|待补充|再参究", "hitPoints": [ { "point": "要点名称", "hit": "full|partial|none", "userPhrase": "用户答到这点的原话片段" } ], "missedPoints": ["要点1", "要点2"], "encouragement": "一句温和鼓励的话，30 字以内", "suggestion": "1-2 句具体补充建议，引导用户看参考答案哪部分", "misunderstanding": "如用户有义理错误，写在这里；无则为空字符串" } 只输出 JSON，不要 ```json 包裹，不要其他文字。

6调用流程
从用户点击提交到看到评分结果的完整链路。
1
用户提交答案
前端收集 user_answer，显示"AI 评分中…"loading
2
后端组装 Prompt
把题目、标准答案、要点、用户答案填入模板
3
调用 MiniMax API
temperature=0.3 保证评分稳定，max_tokens=1500
4
解析 JSON 响应
容错：如解析失败，返回默认反馈
5
前端渲染结果
分色块展示命中/遗漏/建议
6
用户自评掌握度
4 档：不会/模糊/记住/很熟，驱动 SM-2
💻
Python 代码示例
from openai import OpenAI import json client = OpenAI( api_key="your_minimax_key", base_url="https://api.minimaxi.com/v1" ) def grade_open_answer(question, reference, key_points, user_answer): # 组装 Prompt points_text = "\n".join([f" {i+1}. {p}" for i, p in enumerate(key_points)]) user_prompt = f""" 【题目】 {question} 【标准答案】 {reference} 【关键要点】（共 {len(key_points)} 个） {points_text} 【用户作答】 {user_answer} 请按 Prompt 说明的步骤评分，输出 JSON。 """ response = client.chat.completions.create( model="MiniMax-M2.7", messages=[ {"role": "system", "content": SYSTEM_PROMPT}, {"role": "user", "content": user_prompt} ], temperature=0.3, # 低温度保证评分稳定 max_tokens=1500 ) text = response.choices[0].message.content.strip() try: return json.loads(text) except: return {"score": 0, "error": "评分失败，请重试"}
7关键优化要点
上线后可能遇到的问题与对应解决方案。
⚠️
问题 1 · AI 评分过松
症状：用户答了几句就 80 分，严肃修行者觉得没挑战性

解决：

• 调低温度至 0.2

• 在 System Prompt 强调"绝不轻易给 85+ 分除非真正答到 3/4 要点"

• 增加 "strict_mode": true 字段，触发严格评分

⚠️
问题 2 · AI 编造要点
症状：AI 说用户遗漏了"某某要点"，但那不在标准答案里

解决：

• Prompt 明确"只从【关键要点】中选取"

• hitPoints 字段必须完整对应输入要点列表

• 返回后前端验证：输出要点必须在输入中

⚠️
问题 3 · JSON 格式错误
症状：AI 偶尔输出 markdown 包裹或多余文字导致解析失败

解决：

• 后端容错：用正则提取 JSON 块

• 解析失败时自动重试一次

• 二次失败显示友好提示：再试一次

⚠️
问题 4 · 响应慢（> 5 秒）
症状：用户等待时间长，体验差

解决：

• 用 M2.5-highspeed 替代 M2.7

• 开启 streaming，先显示"思考中…"再逐字出反馈

• 前端加入等待动画，配文"观照您的答案…"

8评分数据的进一步用途
AI 评分结果不只是给用户看，还能驱动整个产品的多个功能。
数据用途
score 分数 用户个人统计 · 月度修学报告 · 班级排行榜
hitPoints 命中要点 分析用户对哪些法义掌握好 · 推荐针对性复习
missedPoints 遗漏要点 识别班级整体薄弱点 · 反馈给辅导员 · 下次共修重点讲
misunderstanding 误解 触发"错题本"标记 · 班级层面如多人同错触发公告修正
用户自评掌握度 SM-2 算法输入 · 决定下次复习时间
设计哲学：AI 评分只给建议，不决定命运。用户自评掌握度才是最终驱动复习的决定因素。这样既利用 AI 的效率，又尊重用户自己的学习判断，符合佛法"自证"的精神。
