## 🎯 项目一句话定位

**觉学是一款面向佛教团体的班级共修学习 App**，以《入菩萨行论》为首期课程，采用百词斩式现代 UI + Duolingo 式游戏化机制 + AI 评分思考题 + SM-2 间隔重复算法，让道友跟随班级进度系统修习佛法。

## 🏗️ 核心产品决策（不可轻易变更）

### 1. 定位与差异化

**我们不是"有佛法内容的 App"，而是"让学佛变得好玩的 App"**（引自 Duolingo CEO 哲学）。
- 核心目标：**让本来不会学佛的人，因为好玩而坚持 30 天**
- 核心差异：**班级绑定 + AI 评分思考题**（两个都是其他佛法 App 没有的）

### 2. 首期课程：《入菩萨行论》十品

1. 菩提心利益品
2. 忏悔罪业品
3. 受持菩提心品（当前演示的章节）
4. 不放逸品
5. 护正知品
6. 安忍品
7. 精进品
8. 静虑品
9. 智慧品
10. 回向品

### 3. 班级绑定机制

- 用户通过 **6 位班级邀请码**加入班级
- 班级绑定固定课程表（如 100 天入行论班）
- 章节按**班级进度解锁**（不是用户自选）
- 可见**班级排行榜、道友进度、班级公告**
- 辅导员角色（如扎西仁波切）

### 4. 题型最终定案（12 种）

| # | 题型 | 状态 |
|---|------|------|
| 1 | 单选题（single） | ✅ v1.0 已实现 |
| 2 | 填空题（fill） | ✅ v1.0 已实现 |
| 3 | 连线配对（match） | ✅ v1.0 已实现 |
| 4 | 排序题（sort） | ✅ v1.0 已实现 |
| 5 | 判断多选（multi） | 🆕 v1.0 待实现 |
| 6 | 开放作答（open） | 🆕 v1.0 待实现（核心） |
| 7 | 图片辨识（image） | 🔜 v1.1 |
| 8 | 听颂选答（listen） | 🔜 v1.1 |
| 9 | 卡片翻转（flip） | 🔜 v1.1 |
| 10 | 流程拖拽（flow） | 🔜 v1.2 |
| 11 | 引导分步（guided） | 🔮 v2.0 |
| 12 | 情境判断（scenario） | 🔮 v2.0 |

**v1.0 必做 6 种：** 单选 + 填空 + 连线 + 排序 + 判断多选 + 开放作答
**砍掉的原 v1.0 两种：** 图片辨识（版权成本）/ 听颂选答（录音成本）推迟到 v1.1

### 5. UI 风格

**参考：百词斩（现代明快）**
- 主色：克莱因蓝 `#3B82F6`
- 辅色：橙色 `#F59E0B`、紫色 `#8B5CF6`
- 白色卡片 + 浅阴影 + 圆角 16px
- 立体按钮（底部 3-4px 阴影，按下下沉）
- **抛弃了早期的羊皮纸/古书风格**

### 6. 游戏化机制

- 连续打卡（🔥 火焰图标，也可用 🪷 心灯替代）
- 功德积分（💎 宝石）
- 班级排行榜（周精进榜 🥇🥈🥉）
- 莲花雨完成动画
- 温和错误文案：答错说"再参究"而非"错误"

### 7. 间隔重复（SM-2）

- 四档自评：不会 / 模糊 / 记住 / 很熟
- 间隔：1 / 3 / 7 / 14-30 天
- **AI 评分只给建议，用户自评才驱动 SM-2**（尊重用户判断）

### 8. AI 集成

- **题卡生成：** MiniMax API（M2.7 / M2.5）
- **开放题评分：** MiniMax API，temperature=0.3 求稳定
- **数据源：** 用户提供入行论讲记 + 课后思考题
- **审核：** AI 生成后必须人工（法师）审核才能入库

## 🎯 下一步工作建议（按优先级排序）

### Phase 1：原型完善（1-2 周）

**目标：让原型支持全部 6 种 v1.0 题型**

1. **原型 v4：加入判断多选题**
   - 参考 `question_types_spec.html` 中 `multi` 类型定义
   - UI：checkbox 形式，v1.0 用 strict 判分模式

2. **原型 v5：加入开放作答 + 模拟 AI 评分**
   - 文本输入框 UI
   - 模拟评分反馈（前端 mock 数据）
   - 评分页展示：分数 / 命中要点 / 遗漏要点 / 建议

3. **题型选择器**
   - 右上角加一个"题型演示"选择器，方便展示给辅导员看

### Phase 2：题库编辑器（1 周）

**目标：做一个单页 Web 工具，能快速批量生成真实题库**

- 左侧：粘贴讲记 / 课后思考题
- 中间：点击"AI 生成" → 调 MiniMax
- 右侧：生成的题卡列表，可编辑 / 通过 / 删除
- 底部：通过的题一键导出 JSON

**此步完成后可开始积累真实题库数据。**

### Phase 3：后端启动（2-3 周）

- 选型 Node.js（Express）或 Python（FastAPI）
- 设计核心数据模型：User / Class / Question / UserAnswer / CardState
- 实现 SM-2 调度算法
- 接入 MiniMax API 做开放题评分
- 部署到 Oracle Cloud

### Phase 4：前端开发（3-4 周）

- 选型 React 或 Vue（建议 React + TypeScript）
- 基于原型 v5 的视觉和交互
- 实现所有题型的答题组件
- 实现班级功能、进度页、我的页

### Phase 5：辅导员后台（2 周）

- 班级创建与邀请码生成
- 题库导入与审核
- 班级学员进度查看
- 批量推送公告

## 🔑 重要提示（不能忘）

### 内容相关

1. **版权：** 用户提供的入行论讲记需确认版权状况，商用发布前必须取得授权
2. **法义准确：** AI 生成的题卡必须经法师或熟悉入行论的道友审核
3. **三宝皈依：** 涉及核心法义的题目绝不能让 AI 独立决定对错

### 技术相关

4. **MiniMax API Key：** 不要 commit 到 GitHub，用环境变量或 `.env` 文件
5. **评分稳定性：** 开放题评分用 temperature=0.3，M2.7 模型
6. **数据结构：** 以 `question_types_spec.html` 为唯一标准，所有代码都要遵循

### 产品相关

7. **文案温和：** 佛法 App 不说"失败""错误"，用"再参究""随喜""圆满"
8. **班级是核心：** 不做自由学习模式的 App（早期设想已砍掉）
9. **开放作答是杀手锏：** 其他佛法 App 都没有，必须做好
